## 一、方法设计冻结（第 1 周内必须定稿，后续少改）

### 1.1 明确 Teacher / Student 与任务形式

- **Teacher**：固定为 **Qwen2.5-VL-32B**（具体变体与精度在资源允许范围内选定，但全文只保留 **一个** teacher 配置作为主文结果）。  
- **Student**：固定为 **Qwen2.5-VL-2B**。  
- **任务输出**：在选定标签空间上输出 **类别概率分布**（6 类 / 15 类 / 43 类三套设定可并行，但每一套要有清晰定义）。  
- **蒸馏层级**：**输出层 / 分布层蒸馏**（logit 或概率层面），**第一版不做中间特征蒸馏**（避免不稳定与工程爆炸）。

### 1.2 Teacher 软标签 `p_t` 的获取协议（必须写进附录级细节）

按优先级实现一种主方案，并准备一种备用方案（备用只在主方案失败时启用）：

- **主方案（推荐：概念最清晰）**  
  - 对每张图，构造 **6（或 K）个互斥选项** 的打分方式，使 teacher 输出可归一化为概率向量。  
  - 明确：温度 `T`、解码策略（贪心或低温度采样是否禁止）、**同一协议用于全数据集**（禁止 train/test 换 prompt）。  
- **备用方案（当 teacher 难以直接给 logits 时）**  
  - 用固定 prompt 让 teacher 输出结构化分数或排序，再 **显式归一化** 为 `p_t`，并记录归一化假设。  
- **质量检查**  
  - 随机抽样人工核对：teacher 分布在明显样本上是否合理。  
  - 统计 `p_t` 的香农熵（或集中度）、最大概率、是否退化为 one-hot、是否全均匀。

### 1.3 Student 的 `p_s` 与训练目标（必须能一行写进方法小节）

- Student 对同一张图输出 `p_s = softmax(z_s / T)`（`z_s` 的定义要与 student 模型输出头或“类别打分方式”一致）。  
- **蒸馏项**：  
  - `L_KD = T^2 * KL(p_t || p_s)`（或对称 KL，若选对称必须在全文统一并说明理由）。  
- **监督项**：  
  - `L_CE`：对 ground-truth 的交叉熵（硬标签）。  
- **总目标**：  
  - `L = λ * L_CE + (1 - λ) * L_KD`  
 - **λ、T** 作为消融维度（见第六节）。

### 1.4 LoRA 与蒸馏的组合方式（提前选定2 种主设定）

至少锁定以下两种之一作为主文，另一种放附录或消融：

- **设定 A：LoRA + KD**（通常最符合“参数高效 + 蒸馏”叙事）  
  - Student 主干冻结或部分冻结，LoRA 可训练；蒸馏与 CE 同时作用在 student 的输出分布上。  
- **设定 B：仅 KD（全参数或部分解冻）**  
  - 若算力与稳定性不允许，可弱化或放到附录，但 **不能只有 LoRA 没有 KD**。

### 1.5 风险与回退（团队内部一页纸）

- Teacher 推理过慢 → 预先 **缓存 teacher 软标签**（磁盘格式、版本号、与数据划分绑定）。  
- Student 训练不稳定 → 先 **只 CE收敛**，再逐步打开 KD；或 **KD 权重 warmup**。  
- 显存不足 → teacher 侧用允许的最高效推理配置；**蒸馏训练主要在 student**。

## 二、数据与评测协议升级（与蒸馏设计同步推进）

### 2.1 划分原则（必须可复现）

- **Training**：使用 **官方 train split 的全量或大规模子集**（最低目标：数千张以上；理想：全量）。  
- **Validation**：从 train 中 **独立划分** 或按官方协议外推，但必须在文档中写明随机种子与比例。  
- **Test**：使用 **官方 test split 的更大子集或全量**；禁止与 train 混洗后“假装 test”。  
- **分层信息**：对 6/15/43 类设定，分别报告 **每类样本量**，避免极端长尾被忽略。

### 2.2 标签体系三档（建议并行，但主文聚焦两档）

- **Milestone 档（6 类 coarse）**  
  - 用于对照“从 milestone 到 final 的演进”。  
- **Final 主档（推荐：15 类或更细）**  
  - 定义映射表：原始 43 类 ID → 新粒度；说明合并/拆分原则。  
- **Hard benchmark档（43 类全粒度）**  
  - 即使最终精度一般，也要作为 **难度与真实性** 的证据。

### 2.3 数据工件管理（不写代码，但要列出交付物）

- 每张图的唯一 ID、路径、**三种标签**（若并行）、是否属于 val/test。  
- Teacher 软标签文件：与样本 ID 对齐、记录 teacher 版本与 prompt 版本。  
- 任何子集抽样必须记录 **种子与清单**。

## 三、Teacher 推理与软标签生产流水线（蒸馏落地的“工程心脏”）

### 3.1 环境与资源

- 选定 teacher 运行的机器与精度策略（与课程允许范围一致）。  
- 预估全数据集 teacher 总时长；必要时 **分批运行 + 断点续跑**。

### 3.2 推理批处理与一致性

- 固定：图像分辨率策略、最长生成长度、温度、是否允许自由文本。  
- 对失败样本（解析失败、格式不对）：**重试规则** 与 **剔除比例上限**（写入实验设置）。

### 3.3 校验

- 随机抽样 + 规则检查：概率和为 1、非负、无 NaN。  
- 与少量 **人工可读输出** 对照，确认 teacher 没有系统性偏置（例如总偏向某一类）。

### 3.4 缓存与版本

- 软标签数据集命名包含：**数据划分版本 + teacher 模型名 + prompt 版本 + 标签空间（6/15/43）**。  
- 后续任何 student 实验都 **引用固定版本**，避免“隐形换 teacher”。

## 四、Student 训练（蒸馏 + 可选 LoRA）

### 4.1 最小可行训练闭环

- 小数据过拟合测试：极小 batch上 `L` 是否下降、`p_s` 是否逼近 `p_t`。  
- 再切换到正式训练规模。

### 4.2 日志与可复现

- 记录：学习率 schedule、`λ`、`T`、LoRA rank、可训练参数量、训练时长、硬件。  
- 保存：**best validation metric** 与对应 checkpoint（定义好 metric：macro-F1、accuracy、或 per-class 平均）。

### 4.3 与现有 LoRA 脚本的关系（概念层）

- 明确：哪些模块是 **任务头/打分头**、哪些是 **VLM backbone**。  
- 明确：**蒸馏损失作用在 student 概率输出上**，与 milestone 的 JSON 生成式分类可以并存，但 final 主文建议统一到 **可微的分布监督**（避免解析损失掩盖优化目标）。

## 五、Baseline 矩阵（Final 必须能画成一张表）

### 5.1 必做 baseline（按优先级）

1. **Zero-shot student（2B）**  
2. **Student + LoRA（仅 CE，无 KD）**  
3. **Teacher（32B）直接推理（上界参考）**  
4. **Student + KD（含 LoRA 与否按第四节锁定）**  
5. **传统视觉分类 baseline（ResNet-18 或小型 ViT）**  
   - 输入分辨率与增广策略写清楚；**与 VLM 公平性讨论**可在局限中坦诚说明。

### 5.2 每个 baseline 的统一报告字段

- 整体 accuracy、macro-F1（若类别不平衡）、valid预测成功率（若仍有生成式解析）。  
- **混淆矩阵**、**per-class accuracy**。  
- 推理成本：延迟或 FLOPs/参数的粗对比（至少一句话）。

## 六、消融实验（少而精，但必须成体系）

### 6.1 损失与训练策略

- **LoRA-only vs KD-only vs LoRA+KD**（至少三线）。  
- **λ 扫描**：例如2–3 个代表性取值 + 最优附近一个。  
- **温度 T 扫描**：2–3 个取值，解释对 `p_t`/`p_s` 平滑程度的影响。

### 6.2 数据规模

- **Small vs Large**（例如500 / 2k / full三档中至少两档），回答 **KD 是否在数据稀缺时更有用**。

### 6.3 标签难度

- **6 vs 15 vs 43** 中至少两档对比（建议6 vs 43 或 15 vs 43）。

### 6.4 Teacher 监督类型

- **Hard teacher label vs soft distribution**（若可做）：强化“软标签带来结构信息”的叙事。

### 6.5 LoRA 超参（回应老师批评）

- **Rank r**：在 `{2, 4, 8, 16}` 中至少选 **3 个**做表或曲线。  
- **学习率**：2–3 个数量级或系数。  
- **图像分辨率**：2 档（低/高），说明与速度、精度的权衡。  
- **Prompt 变体**：2 档（仅用于论证鲁棒性，不要散焦）。

## 七、分析内容（Final 深度的主要来源）

### 7.1 定量

- 混淆矩阵对比：**baseline vs LoRA vs KD vs teacher**。  
- Per-class 提升表：**哪些类从 KD 获益最大**（与 teacher 置信度、类内方差关联）。  
- **预测置信度与校准**：student 在 KD 前后是否更校准；可与 teacher 对比。  
- **错误类型**：系统性混淆对（如限速 vs 其他圆形牌）。

### 7.2 定性

- **错误案例图**：展示 KD 修正 zero-shot/LoRA 的典型失败。  
- **Prompt anchoring**：回顾 milestone 现象，说明 fine-tune / KD 如何缓解（需对应数据或统计）。

### 7.3 局限与诚实讨论

- Teacher 计算成本、蒸馏假设、43 类上的瓶颈、与传统 CNN 公平性。  
- 明确 **哪些结论是强证据，哪些是初步观察**。
